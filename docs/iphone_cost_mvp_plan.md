# iPhone 總花費計算器 MVP 規劃

> 作品代號：iphone-cost（暫定）
> 作者：Luck
> 目標：做一個「算你這輩子貢獻蘋果多少錢」的病毒型小作品
> 技術架構：純前端（無需 server）
> 預計開發時間：2–4 天 MVP

---

## 🎯 產品定位

### 一句話定義

> **「輸入你買過的蘋果產品 → 告訴你這輩子貢獻了蘋果多少錢 + 震撼對照 + 一張可分享的圖」**

### 核心價值

- **震撼**：使用者第一次看到「自己花這麼多錢」會想截圖
- **自嘲**：蘋果用戶愛自嘲「我就是韭菜」
- **可分享**：結果頁本身就是一張病毒傳播素材

### 成功條件

使用者看到結果那一刻，**想立刻截圖發 Threads 或 LINE 給朋友**。

## 🧩 MVP 功能拆解

### Step 1：歡迎頁

**主視覺**：
- 標題：**「你這輩子貢獻蘋果多少錢？」**
- 副標：**「30 秒算出你的 Apple 稅」**
- 大按鈕：**「開始計算」**
- 下方小字：
  - ✅ 全程不收集個資
  - ✅ 純前端計算
  - ✅ 結果可分享

**UI 風格**：
- 白底、大留白、蘋果風
- 系統字體（SF Pro / 蘋方 / 思源黑體）
- 大按鈕、高對比
- 有一個 Apple logo 或產品剪影做主視覺

---

### Step 2：選擇你買過的產品

分類呈現，每類獨立分頁或摺疊區塊。

#### 分類 A：iPhone

- 清單：從 iPhone 6（2014）到 iPhone 17 Pro Max（2025）
- 每個型號顯示：
  - 機型名稱
  - 發售年份
  - 起跳價（NT$）
- 可多選（使用者可能擁有多支 iPhone）
- 可選容量（64GB / 128GB / 256GB / 512GB / 1TB，影響價格）

#### 分類 B：Mac

- 機型：MacBook Air、MacBook Pro、Mac mini、iMac、Mac Studio、Mac Pro
- 簡化版：選機型 + 年份 + 配置等級（基本 / 進階 / 頂規）
- 不用太細緻（Mac 配置太複雜，只抓大概）

#### 分類 C：iPad

- 機型：iPad、iPad mini、iPad Air、iPad Pro
- 年份 + 儲存容量
- WiFi vs Cellular 版本（影響價格）

#### 分類 D：穿戴與配件

- AirPods 1 / 2 / 3 / Pro / Pro 2 / Max
- Apple Watch（SE / Series X / Ultra）
- HomePod / HomePod mini
- Apple TV
- AppleCare+（勾選代表有買過）
- **「其他配件打包估算」**：保護殼、充電線、轉接頭、MagSafe 等，讓使用者輸入一個估計數字（預設 NT$5,000–20,000）

#### 分類 E：訂閱服務

- Apple Music（選使用年數）
- iCloud+（選方案 50GB/200GB/2TB + 年數）
- Apple TV+（選年數）
- Apple Arcade（選年數）
- Apple Fitness+（選年數）
- Apple One 組合包

### Step 3：震撼結果頁 ⭐（核心）

這一頁是**整個產品的核心**，必須讓人想截圖。

#### 主要數字（超大字）

```
NT$ 687,420
```
副標：「你這輩子已經貢獻 Apple」

**互動效果**：數字用 count-up 動畫從 0 跳到結果值，增加儀式感。

#### 分區塊內容

**📊 分項統計**
- iPhone：NT$ XXX,XXX（X 支）
- Mac：NT$ XXX,XXX（X 台）
- iPad：NT$ XXX,XXX（X 台）
- 穿戴配件：NT$ XX,XXX
- 訂閱服務：NT$ XX,XXX
- 其他配件：NT$ XX,XXX

**💰 震撼對照**（病毒傳播關鍵）

隨機或依金額大小顯示 3–5 個對照：
- 「= X 台 MacBook Pro M5」
- 「= 環島 X 次（假設每次 5 萬）」
- 「= X 支 Rolex Submariner」
- 「= 台北一房租金 X 年」
- 「= X 張台積電股票（以今日收盤價）」
- 「= Tim Cook 薪水的 X 分鐘」
- 「= 每天不吃早餐 X 年」
- 「= 一台 Toyota Corolla Altis」
- 「= 日本旅遊 X 次（每次 5 萬）」
- 「= iPhone 17 Pro Max X 支」

**📈 如果當年你買的是蘋果股票⋯⋯** ⭐ 殺手級功能

- 假設你第一次買 iPhone 的那一年（例如 2014 買 iPhone 6）
- 如果當時把「所有花在蘋果的錢」一次投進蘋果股票
- 到今天（2026）會值多少？
- 顯示：
  - 原始投入：NT$ 687,420
  - 如果買蘋果股票：NT$ X,XXX,XXX
  - 差距：**NT$ X,XXX,XXX**
  - 幽默註解：「你本可以提早 X 年退休」或「你本可以買一台特斯拉」

**技術實作**：用蘋果歷年股價（寫死 JSON），依使用者「第一次購買蘋果產品的年份」為基準計算複利。

**🏆 你的蘋果用戶等級**（幽默導向）

依總金額分級：
- NT$ 0–100k：**蘋果路人**（還有救）
- NT$ 100k–300k：**蘋果愛好者**（正常用戶）
- NT$ 300k–600k：**蘋果深度使用者**（很認真）
- NT$ 600k–1M：**蘋果信徒**（你是死忠）
- NT$ 1M–2M：**Tim Cook 最愛的人**（想必收到過感謝信）
- NT$ 2M+：**建議你去 Apple 上班**（直接折抵員工價）

每個等級搭配一張 badge 圖 + 幽默評語。

## 🗂️ 資料準備（寫死 JSON）

### 1. Apple 產品售價歷史

**檔案結構**：
```json
{
  "iphone": [
    {
      "id": "iphone-6",
      "model": "iPhone 6",
      "year": 2014,
      "launch_date": "2014-09-19",
      "variants": [
        { "storage": "16GB", "price_twd": 22500 },
        { "storage": "64GB", "price_twd": 25900 },
        { "storage": "128GB", "price_twd": 29500 }
      ]
    },
    // ... iPhone 6s 到 iPhone 17
  ],
  "mac": [...],
  "ipad": [...],
  "wearable": [...],
  "subscription": [...]
}
```

**資料來源**：
- 維基百科「iPhone 機型列表」（有歷代售價）
- Apple 台灣官網歷史公告（archive.org 可查）
- Mobile01 發表時間線文章
- Apple Support 歷代產品頁

**整理時間估計**：2–4 小時整理完全部

**注意**：台幣售價會因匯率有小幅變動，抓當時「上市時台灣官網售價」即可，不用太精確。

### 2. 蘋果股價歷史（年度收盤價，調整過分割）

**檔案結構**：
```json
{
  "AAPL_historical": [
    { "year": 2014, "close_usd": 27.59 },
    { "year": 2015, "close_usd": 26.32 },
    { "year": 2016, "close_usd": 28.95 },
    // ...
    { "year": 2025, "close_usd": 250.00 }
  ]
}
```

**資料來源**：Yahoo Finance 歷史資料可直接下載 CSV（免費）

**計算方式**：
- 使用「調整後收盤價」（adjusted close），已含股票分割
- 匯率可用固定約 30–31 換算台幣（簡化計算）

### 3. 震撼對照物基準金額

**檔案結構**：
```json
{
  "comparisons": [
    {
      "name": "MacBook Pro M5",
      "price_twd": 75000,
      "emoji": "💻",
      "trigger_min": 50000
    },
    {
      "name": "環島一次",
      "price_twd": 50000,
      "emoji": "🏝️",
      "trigger_min": 30000
    },
    {
      "name": "Rolex Submariner",
      "price_twd": 350000,
      "emoji": "⌚",
      "trigger_min": 200000
    },
    {
      "name": "台北一房租金（月）",
      "price_twd": 25000,
      "emoji": "🏠",
      "trigger_min": 100000
    },
    {
      "name": "台積電股票（1 張）",
      "price_twd": 115000,
      "emoji": "📈",
      "trigger_min": 100000
    }
    // ...
  ]
}
```

`trigger_min` 是「當總金額超過這個數字才顯示這個對照」，避免小額使用者看到太誇張的對照。

## 🎨 UI/UX 設計原則

1. **蘋果設計語言**：白底、大留白、SF Pro / 蘋方字體、圓角、柔和漸層
2. **動畫儀式感**：結果數字要用 count-up 動畫、分區塊漸進顯示
3. **手機優先**：80% 流量來自手機，桌面版是加分項
4. **載入速度**：全站首屏 < 2 秒（純前端 + Vercel CDN 很容易達成）
5. **黑色幽默**：文案要幽默但不尖酸，讓使用者想笑不是想生氣
6. **避免 dark pattern**：不要騙使用者繼續選產品（例如預設全勾選）

## 🛠️ 技術棧建議

| 層 | 推薦工具 | 備註 |
|---|---|---|
| 框架 | Next.js 15（App Router） | 也可用 Astro 更輕 |
| 樣式 | Tailwind CSS | 快速開發 |
| 狀態管理 | React useState 或 Zustand | 不需要 Redux |
| 分享圖生成 | `satori` (Vercel OG) 或 `html-to-image` | satori 產生更快、更穩 |
| 動畫 | `framer-motion` + `react-countup` | 數字跳動效果 |
| 部署 | Vercel（免費方案夠用） | GitHub 連動自動部署 |
| 分析 | Vercel Analytics + Google Analytics | 雙保險 |
| 字體 | 系統字體 + Google Fonts（思源黑體） | 不用自己 host |

**關於語言**：Next.js 15 App Router 適合這種小型純前端專案，SEO 友善。

**關於資料**：所有 JSON 放 `/data` 資料夾，直接 import 進來用，不需要 API。

## 📊 成功 / 失敗判斷

### 第 14 天檢視

**爆紅訊號**（任一達成就算成功）：
- 日均 UV > 1,000
- 有人自發分享到 IG / Threads（搜得到）
- 開始有反向連結（被部落格或新聞提到）
- 信用卡 affiliate 開始有點擊

**溫和成長**：
- 日均 UV 100–500
- 有穩定 SEO 流量進來
- 幾個主動分享但沒破圈

**沒起來**：
- 日均 UV < 100
- 只有你自己 + 家人朋友在用
- 沒有 organic 分享

### 應對策略

**爆紅** → 立刻：
- 買獨立 domain 做 301 轉址
- 加英文版
- 聯絡信用卡業務談更高分潤
- 規劃「Google 花費」「Amazon 花費」續作

**溫和成長** → 持續：
- SEO 優化
- 寫部落格文章導流（「蘋果用戶的集體焦慮」之類）
- 做第 2 個作品分散風險

**沒起來** → 接受：
- 把專案當作品集（加到履歷 / GitHub）
- 寫一篇「我做了 XX 但失敗了」的文章，本身可能會紅
- 立刻做下一個題目，不要糾結

## 🎯 關鍵成功要素（排序重要性）

1. **分享卡片設計** ⭐⭐⭐⭐⭐
   - 這是爆紅最關鍵的因素
   - 卡片醜陋 = 沒人分享 = 不紅
   - 值得花 30% 的開發時間在這

2. **震撼對照文案** ⭐⭐⭐⭐⭐
   - 對照物要夠「有感」、夠「震撼」
   - 文案要幽默不尖酸
   - 這決定使用者「想不想截圖」

3. **蘋果股票對比** ⭐⭐⭐⭐
   - 這是「殺手級的病毒因素」
   - 「本可以提早退休」這種文案超有傳播力

4. **UI 設計美感** ⭐⭐⭐⭐
   - 蘋果用戶對設計很挑
   - 醜 = 沒說服力

5. **手機 RWD** ⭐⭐⭐⭐
   - 80% 流量從手機來
   - 手機沒做好直接失敗

6. **資料完整性** ⭐⭐⭐
   - iPhone 資料要完整（從 iPhone 6 到最新）
   - 股價資料要正確（計算錯誤會被打臉）

7. **推廣策略** ⭐⭐⭐
   - 好的作品會自己紅
   - 但前 100 個使用者還是要手動推

## 💡 題目命名的兩種思路

### 選項 A：「iPhone 總花費」（聚焦）
- **優點**：更病毒（80% 人只想知道 iPhone 花了多少）
- **優點**：名字精準
- **缺點**：金額較小、沒那麼震撼

### 選項 B：「Apple 總花費」（完整）
- **優點**：金額更大、更震撼
- **優點**：更完整（Mac / iPad / 配件 / 訂閱）
- **缺點**：使用者要選更多東西，流程較長

### 建議

**做 Apple 總花費（完整版）**，但**首頁標題用 iPhone 吸引點擊**：
- 標題：「你這輩子貢獻蘋果多少錢？」
- 副標：「從 iPhone、Mac 到訂閱服務全算上」
- 流程中使用者可以「跳過不相關分類」快速得到結果

這樣既有震撼金額，又有易上手體驗。
